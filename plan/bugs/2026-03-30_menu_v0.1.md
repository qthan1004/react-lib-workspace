# Menu Component Bug Report

Based on testing the new Menu features in Storybook, I have identified several major functional and visual bugs primarily concerning the new `mode="popover"` and `display="icon"` features. 

Below is the breakdown of issues that need to be addressed.

## 1. Popover Mode Functions as Inline (Critical)
When setting `<Menu mode="popover">`, the `MenuSub` components do not render as floating popovers. Instead, they expand downwards inline, pushing the rest of the menu items down. Functionally and visually, there is currently no difference between `inline` and `popover` modes. 

![Popover functions inline](/home/administrator/.gemini/antigravity/brain/77d2b5da-160b-4fe7-b814-fd42807dab6f/popover_submenu_hover_1774866580584.png)

## 2. Hover Trigger Does Not Open Sub-menu
In the "Popover Sub-menus" story, hover triggers do not work. Hovering over a `MenuSubTrigger` does not open the floating sub-menu. The sub-menu only expands when clicking on it (and even then, it expands inline, as noted above).

## 3. Incorrect Caret Icon in Popover Mode
For items that have a sub-menu, the caret currently always points downwards (`▾`). If a sub-menu is supposed to fly out to the right (as a floating popover), the caret should ideally point right (`▸`) to indicate the direction it will open.

## 4. `display="icon"` Does Not Hide Text Labels
Setting `<Menu display="icon">` fails to hide the text labels of the `MenuItem`s. The menu retains its full standard width and displays both the icon and the text label, entirely failing the purpose of the "Icon-Only Display (Mini Sidebar)".

![Mini sidebar failing to hide text](/home/administrator/.gemini/antigravity/brain/77d2b5da-160b-4fe7-b814-fd42807dab6f/mini_sidebar_click_1774866620528.png)

## 5. Visual Separation (Addressed in Storybook)
Initially, it was hard to visually distinguish the inline mode from the popover mode in the story. **Note:** This has been resolved by placing them inside styled `<aside>` blocks with colored backgrounds for clarity in the newly updated Storybook code.

## 6. CSS/Styled-Component Issues
Because labels aren't hidden and sub-menus lack a proper floating portal/reference setup, there seem to be missing implementations in the layout styled components (e.g. `MenuItemLabelStyled`, internal Floating UI usage) connecting the props `display="icon"` and `mode="popover"` down to the DOM.

> [!WARNING]
> It appears that the recent implementation of `03-menu-sub-resolve-mode-floating.md` added the props and types for these features, but the actual CSS logic (hiding labels in icon mode) and Floating UI logic (rendering floating portals for `MenuSubContent` when in popover mode) is either missing, disconnected, or failing to trigger.

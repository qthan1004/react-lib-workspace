## 🔴 Critical (Feature hoàn toàn hỏng)

### 1. Keyboard navigation does not change slider value
* **Status:** Open
* **Story:** `Basic` — http://localhost:6006/iframe.html?id=slider-slider--basic&viewMode=story&full=1
* **Screenshot:** `plan/test-logs/screenshots/slider-basic-keyboard-bug.png`
* **File:** `libs/slider/src/lib/Slider/Slider.tsx`
* **Description:** Pressing ArrowRight/ArrowLeft when slider thumb is focused does not update the value. The `onKeyDown` handler is attached to `SliderThumbStyled` (line 191) which requires the thumb element to be focused directly. However clicking the container (`SliderRootStyled`) does not transfer focus to the thumb — `containerRef.current?.focus()` (line 65) focuses the root div, not the thumb.
* **Steps to reproduce:**
  1. Navigate to slider Basic story
  2. Click the slider track
  3. Press ArrowRight multiple times
* **Expected:** Value increases by `step` each press (30 → 31 → 32...)
* **Actual:** Value stays at 30. Keyboard input has no effect.

## 🟡 Major (Hành vi sai / thiếu kết nối)

### 2. Vertical slider missing aria-orientation attribute
* **Status:** Open
* **Story:** `Vertical` — http://localhost:6006/iframe.html?id=slider-slider--vertical&viewMode=story&full=1
* **Screenshot:** `plan/test-logs/screenshots/slider-vertical-aria-bug.png`
* **File:** `libs/slider/src/lib/Slider/Slider.tsx`
* **Description:** The `SliderThumbStyled` element with `role="slider"` does not set `aria-orientation`. For vertical sliders, this means screen readers report the slider as horizontal (browser default). Should add `aria-orientation={orientation}` to the thumb element.
* **Steps to reproduce:**
  1. Navigate to Vertical story
  2. Inspect the slider thumb element's ARIA attributes
* **Expected:** `aria-orientation="vertical"` on the slider thumb
* **Actual:** No `aria-orientation` attribute; defaults to `horizontal`

### 3. Controlled slider click on track does not update value
* **Status:** Open  
* **Story:** `Controlled` — http://localhost:6006/iframe.html?id=slider-slider--controlled&viewMode=story&full=1
* **File:** `libs/slider/src/lib/Slider/Slider.tsx`
* **Description:** Clicking the slider track in the Controlled story does not visually change value (stays at 50). The `handlePointerDown` calls `handleValueChange` which calls `onChange` but the story's `setValue` may not be re-rendering due to pointer capture or event issues. Needs investigation.
* **Steps to reproduce:**
  1. Navigate to Controlled story
  2. Click anywhere on the track
* **Expected:** Value changes to the clicked position, "Value: XX" text updates
* **Actual:** Value stays at 50

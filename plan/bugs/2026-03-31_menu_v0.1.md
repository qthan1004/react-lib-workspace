# Menu Library Bug Report

**Date:** 2026-03-31
**Module:** `menu`
**Version:** `v0.1`

## Open Issues

### 1. Nested Popover Submenu Not Working (Critical)
* **Status:** ❌ Confirmed Broken
* **Description:** Inside a Floating Popover (e.g. Story 10 - Right Side), clicking or hovering on a sub-menu trigger (like "More Data ▸") causes the parent popover to close immediately. The nested popover never appears. Likely an issue with FloatingTree, context inheritance, or lost focus constraints.

### 2. Icon-Only Display Mini Sidebar Popover (Major)
* **Status:** ❌ Confirmed Broken
* **Description:** In the Icon-Only Display story, hovering over sidebar icons (🏠, 👤, 📊) does nothing. Clicking an icon opens an empty floating popover box without any text or icon content properly visible inside it. Trigger mechanisms and context passing to the sub-items are failing in compact mode.

### 3. React State Update Warning (Technical Debt)
* **Status:** ⚠️ Needs Investigation
* **Description:** Console throws multiple warnings: `Cannot update a component ('MenuSubInline') while rendering a different component ('MenuSubInline')`. This usually points to circular updates or setting state directly during the render cycle (likely in `registerSelected` inside `MenuSub.tsx` or `MenuItem.tsx`).

### 4. Popover Hover Trigger Inconsistency (Minor)
* **Status:** ⚠️ Needs Investigation
* **Description:** In the `Popover Triggers & Settings` test story, hovering over the first-level trigger ("Analytics") often fails to reliably open the popover. The hover interaction area seems unresponsive or detached. Conversely, the `trigger="click"` setting works reliably with correct placements and offsets.

## 1️⃣ Extract Keyboard Logic → `libs/utils` ✅ DECIDED

### Impact: 🔴 HIGH — Cross-library, reusable foundation

### Scope cho Menu Plan

Trong scope của **menu**, chỉ cần:

1. **Tách `useMenuKeyboard`** thành 2 phần:
   - Generic keyboard handling → `libs/utils/hooks/useKeyboard.ts`
   - Menu-specific typeahead → giữ trong `libs/menu/hooks/useMenuTypeahead.ts`
2. **Move `cancelEvent`** → `libs/utils/helpers/cancelEvent.ts` (general utility)
3. **`useMenuKeyboard`** trở thành thin wrapper composing `useKeyboard` + typeahead

### File Changes (Menu scope)

```
libs/utils/src/lib/
  hooks/
    useKeyboard.ts           ← extract generic keyboard handler
  helpers/
    cancelEvent.ts           ← moved from menu/helpers

libs/menu/src/lib/hooks/
  useMenuKeyboard.ts         ← refactor: compose useKeyboard + typeahead
  useMenuTypeahead.ts        ← extract typeahead logic (menu-specific)
```

### Áp dụng cho Menu

```typescript
// useMenuKeyboard.ts — sau khi refactor
import { useKeyboard } from '@thanh-libs/utils';
import { useMenuTypeahead } from './useMenuTypeahead';

const menuCommands: KeyboardCommand[] = [
  { key: 'ArrowDown', execute: (ctx) => ctx.focusRelative(+1) },
  { key: 'ArrowUp',   execute: (ctx) => ctx.focusRelative(-1) },
  { key: 'Home',      execute: (ctx) => ctx.focusFirst() },
  { key: 'End',       execute: (ctx) => ctx.focusLast() },
];

export const useMenuKeyboard = (containerRef) => {
  const typeaheadCommand = useMenuTypeahead(containerRef);
  return useKeyboard([...menuCommands, typeaheadCommand], {
    containerRef,
    getItems: () => getVisibleMenuItems(containerRef.current),
  });
};
```

### Deep Design → Separate Plan

> [!IMPORTANT]
> Chi tiết kiến trúc `useKeyboard` (Command Pattern interfaces, **Scope support** cho hotkeys nested form/section, `KeyboardContext`, `KeyboardCommand` types) sẽ được thiết kế trong **plan riêng**: `plan/utils/useKeyboard`.
>
> Các yêu cầu đã ghi nhận cho plan đó:
> - Command Pattern với `key`, `modifiers`, `execute`, `enabled`, `stopPropagation`
> - **Scope support** via DOM event bubbling (inner scope xử lý trước)
> - `KeyboardContext` với focus helpers (`focusRelative`, `focusFirst`, `focusLast`)
> - Extensible cho Form, TreeView, Combobox

### Complexity & Risk

| Aspect | Assessment |
|--------|-----------|
| Complexity | **Low-Medium** — Extract + compose, behavior không đổi |
| Breaking change | **None** — `useMenuKeyboard` API giữ nguyên |

## 2️⃣ Focus Trap Outline — Chỉ cần Accessibility, không cần Visual

### Impact: 🟡 MEDIUM — UX polish, quick win

### Phân tích hiện trạng

Hiện tại [styled.tsx:90-93](file:///d:/workspace/react-lib-workspace/libs/menu/src/lib/styled.tsx#L90-L93):

```typescript
'&:focus-visible': {
  outline: '2px solid transparent',
  boxShadow: `inset 0 0 0 2px ${focusRingColor}`,
},
```

Đây là focus ring cho **individual MenuItem** — đã OK vì dùng `focus-visible` (chỉ hiện khi keyboard navigate, không hiện khi click).

Vấn đề bạn nói có thể từ **Menu container** ([Menu.tsx:55](file:///d:/workspace/react-lib-workspace/libs/menu/src/lib/components/Menu.tsx#L55)):

```tsx
<MenuContainerStyled
  tabIndex={0}  // ← Cái này tạo outline khi focus container
  ...
>
```

Container có `tabIndex={0}` → browser tự thêm default outline khi container nhận focus.

### Đề xuất Fix

```typescript
// MenuContainerStyled — thêm:
'&:focus': {
  outline: 'none',
},
// Không cần focus-visible cho container vì nó chỉ là "entry point"
// Focus sẽ ngay lập tức chuyển sang item đầu tiên (useMenuKeyboard handleFocusIn)
```

**Accessibility vẫn đảm bảo** vì:
- Container có `role="menu"` + `tabIndex={0}` → screen reader vẫn announce
- Focus ngay lập tức chuyển sang child item (via `handleFocusIn` trong `useMenuKeyboard`)
- Child items có `&:focus-visible` ring riêng

### Complexity & Risk

| Aspect | Assessment |
|--------|-----------|
| Complexity | **Very Low** — 2-3 lines CSS |
| Breaking change | **None** |
| Risk | **None** — Container focus is instantly delegated to items |
| Files changed | 1 file (styled.tsx) |

## 3️⃣ Hover/Select Visual Hierarchy + Inline Dot + Active Icon

### Impact: 🔴 HIGH — Core visual design, most visible to users

### Phân tích hiện trạng

#### Current hover behavior (styled.tsx):
```
Parent item:  hover → hoverBg + hoverColor change ✓
Child item:   hover → same hoverBg + hoverColor change (identical to parent)
Selected:     activeBg + fontWeight: 600 + ✓ check bên trái
SoftSelected: softSelectedBg (parent has selected child)
Active icon:  ❌ không có — chỉ có check mark bên trái
```

Vấn đề: **Parent và child hover giống nhau** → Thiếu visual hierarchy. Khi menu có nhiều level, user khó phân biệt level nào đang interact.

### Đề xuất Design

#### A. Hover Differentiation: Parent vs Child ✅ DECIDED

```
┌─────────────────────────────────────────────────┐
│ PARENT items (top-level, hoặc SubTrigger):      │
│   hover → background + chữ đổi màu             │
│           (giữ nguyên như hiện tại)              │
│                                                 │
│ CHILD items (trong SubContent):                 │
│   hover → tương tự select nhưng nhạt hơn        │
│           background nhẹ + text change           │
└─────────────────────────────────────────────────┘
```

**Implementation approach:**

Cần phân biệt "depth level" — MenuItem biết mình là child hay parent. Hai cách:

**Option A: CSS-only via nesting selector** *(Recommended — zero JS change)*
```typescript
// InlineSubContentStyled children → lighter bg (like select but softer)
'& [role="menuitem"]': {
  '&:hover': {
    backgroundColor: childHoverBg,   // subtle bg (lighter than select)
    color: childHoverColor,          // text brightens
  },
}
```

**Option B: Context-based prop** *(More explicit but more refactoring)*
```typescript
// MenuItem nhận isChild prop từ SubContent context
<MenuItemStyled ownerIsChild={isInsideSubContent} ... />
```

> [!TIP]
> **Đề xuất Option A** — CSS nesting giữ MenuItem API không đổi, zero breaking change. CSS specificity xử lý hết.

#### B. Select Intensity Hierarchy

```
┌────────────────────────────────────────────────────┐
│ hover:          nhẹ nhàng (light bg hoặc chỉ text) │
│ softSelected:   bg nhạt (parent có child selected)  │
│ selected:       bg đậm hơn + bold + optional color  │
│                                                      │
│ Ordering: hover < softSelected < selected            │
└────────────────────────────────────────────────────────┘
```

Hiện tại đã có logic này, nhưng cần tuning:

```typescript
// Hiện tại:
const selectedBg = 'rgba(25,118,210,0.08)';    // rất nhạt
const softSelectedBg = 'rgba(0,0,0,0.04)';     // cũng nhạt

// Đề xuất tăng contrast:
const selectedBg = 'rgba(25,118,210,0.16)';    // đậm hơn rõ ràng
const softSelectedBg = 'rgba(25,118,210,0.06)'; // nhạt nhẹ nhàng
const hoverBg = 'rgba(0,0,0,0.04)';             // lightest
```

#### C. Inline Dot Indicator

```
┌──────────────────────────────────┐
│ 📊 Analytics              ▾     │  ← parent (SubTrigger)
│   • Overview                     │  ← dot before child items
│   • Reports           ← active  │  ← dot filled khi active
│   • Exports                      │
│ ⚙ Settings              ▾     │
│   • General                      │
│   • Security                     │
└──────────────────────────────────┘
```

**Implementation:**

Thêm optional `showDot` prop vào `MenuSubContent` (hoặc context level):

```typescript
// InlineSubContentStyled — thêm dot pseudo-element cho children
'& > [role="menuitem"]': {
  '&::before': {
    content: '""',
    display: 'inline-block',
    width: 6,
    height: 6,
    borderRadius: '50%',
    border: `1.5px solid ${dotColor}`,
    backgroundColor: 'transparent',
    marginRight: spacing?.small ?? '0.5rem',
    flexShrink: 0,
    transition: 'background-color 150ms, border-color 150ms',
  },
  // Hover: fill dot
  '&:hover::before': {
    backgroundColor: dotColor,
    borderColor: dotColor,
  },
  // Selected: fill dot with active color
  '&[aria-current="page"]::before': {
    backgroundColor: activeColor,
    borderColor: activeColor,
  },
}
```

**Prop design:**

```typescript
// Option 1: On MenuSubContent (inline only)
<MenuSubContent showDot>
  <MenuItem>Overview</MenuItem>
</MenuSubContent>

// Option 2: On Menu root (global)
<Menu inlineDot>...</Menu>

// Option 3: On MenuSub (per submenu)
<MenuSub showDot>...</MenuSub>
```

> [!IMPORTANT]
> Bạn nói "optional chỗ dot" — vậy nên là **prop** chứ không phải mặc định. Đề xuất đặt trên `MenuSub` hoặc `MenuSubContent` vì chỉ áp dụng cho inline mode. Cũng có thể thêm `inlineDot` trên `Menu` root để global toggle.

#### D. Active Icon Indicator (Trailing — Parent Items Only) ✅ DECIDED

**Decisions locked in:**
- ✅ **Global API** trên `Menu` root — không phải per-item
- ✅ **Mặc định BẬT** — user tắt bằng `activeIndicator={false}`
- ✅ **2 built-in variants**: `'dot'` (default) và `'bar'`
- ✅ **Chỉ hiện khi selected** — hover KHÔNG hiện indicator
- ✅ **States**: outlined (normal/not selected) → filled (selected)
- ✅ Custom ReactNode override vẫn được

**Visual — indicator chỉ hiện khi selected:**
```
┌────────────────────────────────────────────────────────┐
│ Not selected:     (nothing)   ← không hiện gì         │
│ Hover:            (nothing)   ← hover cũng không hiện │
│ Selected:         ●  (filled dot/bar, active color)    │
│                                                        │
│ Bar variant:      ▌ (filled bar khi selected)          │
└────────────────────────────────────────────────────────┘
```

**Visual in context:**
```
┌───────────────────────────────────────┐
│ 🏠 Dashboard                ●       │  ← selected → filled dot (active color)
│ 📊 Analytics                ▾       │  ← SubTrigger → arrow, NO indicator
│   • Overview                         │  ← dot luôn hiện (outlined → filled)
│   • Reports                          │
│ 👤 Profile                          │  ← not selected → KHÔNG hiện indicator
│ ⚙ Settings                          │  ← normal → KHÔNG hiện indicator
└───────────────────────────────────────┘
```

### Final API Design

```typescript
// Type definition
export type MenuActiveIndicator = 'dot' | 'bar' | false | ReactNode;

// Menu props
export interface MenuProps {
  // ... existing ...
  /** 
   * Active indicator on selected items. 
   * Default: 'dot' (shown on all selected MenuItems)
   * - 'dot': outlined dot → filled on hover/active
   * - 'bar': vertical bar → filled on hover/active
   * - false: disable indicator
   * - ReactNode: custom indicator element
   */
  activeIndicator?: MenuActiveIndicator;
}
```

**Usage examples:**
```tsx
// Default behavior — dot indicator ON
<Menu>
  <MenuItem selected>Home</MenuItem>     {/* → ● filled dot */}
  <MenuItem>Profile</MenuItem>           {/* → nothing */}
</Menu>

// Explicit variant
<Menu activeIndicator="bar">
  <MenuItem selected>Home</MenuItem>     {/* → ▌ filled bar */}
</Menu>

// Turn off
<Menu activeIndicator={false}>
  <MenuItem selected>Home</MenuItem>     {/* → no indicator */}
</Menu>

// Custom icon
<Menu activeIndicator={<MyCustomIcon />}>
  <MenuItem selected>Home</MenuItem>     {/* → custom icon */}
</Menu>
```

### Implementation — Built-in Indicator Component

```typescript
// Tạo internal component cho 2 built-in variants

// Dot variant (default)
const DotIndicator = styled.span<{ ownerActive: boolean; ownerColorScheme?: MenuColorScheme }>(
  ({ ownerActive, ownerColorScheme }) => {
    const { palette } = useTheme();
    const indicatorColor = ownerColorScheme?.activeIconColor 
      ?? palette?.primary?.main ?? '#1976d2';
    
    return {
      display: 'inline-block',
      width: 6,
      height: 6,
      borderRadius: '50%',
      border: `1.5px solid ${indicatorColor}`,
      backgroundColor: ownerActive ? indicatorColor : 'transparent',
      transition: 'background-color 150ms, border-color 150ms',
    };
  }
);

// Bar variant
const BarIndicator = styled.span<{ ownerActive: boolean; ownerColorScheme?: MenuColorScheme }>(
  ({ ownerActive, ownerColorScheme }) => {
    const { palette } = useTheme();
    const indicatorColor = ownerColorScheme?.activeIconColor 
      ?? palette?.primary?.main ?? '#1976d2';
    
    return {
      display: 'inline-block',
      width: 3,
      height: 16,
      borderRadius: 1.5,
      border: `1px solid ${indicatorColor}`,
      backgroundColor: ownerActive ? indicatorColor : 'transparent',
      transition: 'background-color 150ms',
    };
  }
);
```

### Render Logic trong MenuItem

```typescript
// MenuItem.tsx — resolve indicator
const { activeIndicator } = useMenuContext();  // global config

// Render trailing indicator — CHỈ hiện khi selected
const renderIndicator = () => {
  if (!selected || activeIndicator === false || isIconOnly) return null;
  
  // Custom ReactNode
  if (activeIndicator !== 'dot' && activeIndicator !== 'bar' && activeIndicator !== undefined) {
    return (
      <MenuItemIndicatorStyled>{activeIndicator}</MenuItemIndicatorStyled>
    );
  }
  
  // Built-in variants — CHỈ filled (vì chỉ hiện khi selected)
  const variant = activeIndicator ?? 'dot';  // default 'dot'
  const IndicatorComponent = variant === 'bar' ? BarIndicator : DotIndicator;
  
  return (
    <MenuItemIndicatorStyled>
      <IndicatorComponent ownerActive={true} ownerColorScheme={colorScheme} />
    </MenuItemIndicatorStyled>
  );
};
```

> [!NOTE]
> **Simplified logic**: Vì indicator chỉ hiện khi `selected`, `ownerActive` luôn là `true` → indicator luôn filled. Không cần CSS hover state.

**Tại sao không cần handle cho `MenuSubTrigger`?**

`MenuSubTrigger` là component riêng biệt ([MenuSubTrigger.tsx](file:///d:/workspace/react-lib-workspace/libs/menu/src/lib/components/MenuSubTrigger.tsx)) — nó **không dùng `MenuItem`**, mà dùng `MenuItemStyled` trực tiếp với `SubArrowStyled` ở trailing position. Vậy indicator logic chỉ nằm trong `MenuItem.tsx`, tự nhiên không conflict.
